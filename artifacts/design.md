# 世界杯每日赛程预测 H5 设计稿说明

## 1. 设计目标
- 为嵌入直播吧 App 的 mobile-web 活动页提供可评审高保真方案，覆盖首页、达人榜与核心状态。
- 用“世界杯高亮主题”建立赛事氛围：高饱和金绿主色、冷暖交叠光晕、竞技感排版、通透卡片材质。
- 保证低成本理解与高转化操作，用户在 3 秒内能完成“看赛程、点预测、看分布、去榜单”的主路径识别。

## 2. 信息架构
### 2.1 活动首页
1. 顶部 KV 区
2. 战绩摘要条
3. 达人榜入口卡
4. 日期切换器
5. 赛程状态筛选 Chips
6. 比赛卡片列表
7. 规则说明与底部氛围收束区

### 2.2 达人榜页
1. 榜单头部与返回入口
2. 每日榜 / 总榜 Tab
3. TOP3 焦点 podium
4. TOP4-TOP20 列表
5. 固定底部“我的排名”条

## 3. 视觉方向
### 3.1 主题关键词
- 世界杯高亮
- 玻璃拟态
- 微渐变
- 冠军金属感
- 赛场灯光氛围

### 3.2 色彩
- 背景主色：深墨绿 `#081c16` 到墨蓝绿 `#0c2b24`
- 高亮金：`#f8d978`
- 霓虹绿：`#7df2ae`
- 功能蓝：`#85d8ff`
- 警示橙：`#ffb35c`
- 取消灰：`#96a7aa`
- 命中红：`#ff6d5f`

### 3.3 材质与层次
- KV 与页面背景使用同一组纵向渐变，避免滚动时出现断层。
- 战绩条与首张功能卡使用 `-48px` 负边距搭叠 KV，形成头部压层关系。
- 卡片采用 12% 到 20% 透明白叠层、1px 高光描边、轻微模糊背景与内阴影。
- CTA 使用金色渐变 + 内发光 + 投影，保持“按钮像可按下的奖章”质感。

### 3.4 字体与排版
- 中文主字体建议：`"DIN Alternate", "PingFang SC", sans-serif`
- 数字与比分突出使用更粗字重与字距压缩
- 标题采用大号字与大写英文点缀，增强赛事海报感

## 4. 页面说明
### 4.1 首页
- KV 文案聚焦“今日赛程预测”，附副标题“命中越多，榜单越高”。
- 战绩摘要显示总命中、昨日战绩、击败比例三个指标。
- 达人榜入口卡放在首屏中段，使用 podium 缩略预览引导进入榜单页。
- 日期条默认高亮“今天”，左右保留过去 / 未来日期；无赛事日期显示禁用态。
- 列表区域支持通过状态筛选快速切换样例，用于设计评审展示完整状态机。

### 4.2 比赛卡片通用结构
- 头部：轮次、开赛时间、状态胶囊、详情入口
- 中部：主客队、比分或 VS、氛围标签
- 预测区：胜平负、总进球、球员进球三类题
- 底部：分布 / 结算 / 异常说明 / CTA

## 5. 状态设计
### 5.1 可预测
- 选项可点击，未展示全站分布。
- 底部主 CTA 显示“去赛事详情”，预测按单题即时提交。

### 5.2 已预测
- 已选选项为高亮金色，未选项降低透明度。
- 每个题块下方展开全站分布条，并显示“你的选择”标签。

### 5.3 已锁定
- 交互禁用，若用户已参与则显示“等待开奖”，未参与则显示“预测已截止”。
- 比赛信息区增强“进行中 / 即将开赛”状态提醒。

### 5.4 已结算
- 展示正确答案、我的结果、命中/未中标识、击败百分比。
- 比分区替换为终场比分，结算信息收纳在底部高亮区。

### 5.5 已取消
- 使用去饱和处理与斜纹遮罩，弱化竞技高光。
- 文案明确“本场取消，相关题目不计入成绩”。

### 5.6 无赛事
- 日期区仍保留，但列表改为空态插画风模块。
- 通过延展背景光晕、规则说明和明日赛程预告避免底部空白。

## 6. 达人榜设计
- 榜单页沿用首页背景体系，但头部改为更集中式聚光灯效果。
- 每日榜强调“今日命中数”，总榜强调“累计命中 + 命中率”。
- TOP3 使用奖台结构，冠军位更高并带金色圆环。
- 固定底条显示“我的排名 / 命中题数 / 与上一名差距”，强化回访动机。

## 7. 交互说明
- 首页顶部导航切换“活动首页 / 达人榜”。
- 日期标签、状态筛选、榜单 Tab 均有动效下划条或底光移动。
- 比赛卡片中的预测项点击后，静态 mock 数据即时切换到已预测态。
- 状态样例支持通过 Demo 控件切换，便于 design review。

## 8. 适配与实现备注
- 画布按 390px 视觉宽度设计，真实页面宽度采用 `min(100vw, 430px)` 容器。
- 底部通过背景扩展层、规则卡和安全区 padding 保证无大面积空白。
- 分享战绩卡已从范围移除，mockup 不包含该页面。
- 所有内容基于静态数据演示，不依赖真实接口。

## 9. 评审关注点
- KV 与页面背景是否足够无缝
- 战绩条负边距搭叠是否成立
- 六类状态的语义是否一眼可懂
- 底部填充策略是否解决低赛程日空白问题
- 榜单页是否形成独立但统一的视觉记忆
