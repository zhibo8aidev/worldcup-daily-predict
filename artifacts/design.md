# 世界杯每日赛程预测 H5 高保真设计说明 v3

## 1. 设计结论
- 本版为 `mobile-web / H5` 评审稿，不采用原生 App 导航结构。
- 所有产物已按最新批准 PRD 与最新设计规范重做，旧版口径全部覆盖。
- 设计主轴为：`世界杯高亮主题 + 高阶 3D KV + 玻璃拟态卡片 + 负边距搭叠 + 统一提交模型 + 底部防空白`.

## 2. 本版必须体现的最终口径
1. 提交规则以 PRD `3.2.3 / 3.3.1` 为准：单场三题答完后统一点击 `提交预测` 提交。
2. `提交预测` 按钮文案始终不变，不出现“已提交 / 查看复盘 / 已锁定”等替换文案，仅通过高亮、置灰、禁用反馈体现状态。
3. 焦点球员数量按 PRD `3.2.2` 的“若干名焦点球员”处理，前端支持 1 组至多组球员对比或降级说明。
4. 首页右上角固定悬浮入口：`活动规则`、`预测达人榜`。
5. 个人战绩摘要条最左侧展示用户头像。
6. 对阵头像与比赛时间区域均为比赛综合内页入口。
7. 分享战绩卡在交付范围内，且在 mockup 中可直接切换查看。
8. KV 头图采用高阶 3D 建模感表达，背景与正文无缝衔接。

## 4. 视觉系统
### 4.1 主题关键词
- 世界杯高亮
- 体育盛典
- 商业插画级 3D
- 玻璃拟态
- 奖章金属质感

### 4.2 色彩定义
- 主背景深绿：`#0b2b26`
- KV 过渡绿：`#184d40`
- 高亮金：`#f6e4a6`
- 金属琥珀：`#cf8f2c`
- 冰蓝高光：`#8ddfff`
- 荧光草绿：`#77f1b8`
- 告警橙：`#ffb15f`
- 结算红：`#ff6a5c`
- 取消灰：`#95a7ae`

### 4.3 材质与空间
- KV 与页面背景采用连续同色系纵深渐变，保证滑动时无断层。
- 战绩摘要条采用 `-52px` 负边距搭叠在 KV 下缘。
- 赛程卡、榜单卡、分享卡使用玻璃拟态底板与细描边高光。
- CTA、Tab、日期胶囊与选项按钮使用微渐变和内发光，不使用生硬纯色块。
- 页面尾部使用环境渐变、赛事氛围文案和规则卡做“压舱”，避免低赛程日出现大片空白。

### 4.4 KV 主视觉要求
- 视觉中心由奖杯、足球、光环、飘带与球场灯架构成。
- 标题需具备体积感与光效，不使用纯黑描边。
- KV 调性需偏赛事商业海报，而非低幼卡通。

## 5. 首页设计说明
### 5.1 KV 区
- 宽度按 `390px` 视觉稿铺满容器。
- 主标题：`今日赛程预测`
- 副标题：`单场三题答满再提交，命中越多越靠前`
- 顶部加入品牌位与赛事阶段信息，强化活动权威感。

### 5.2 右上角悬浮入口
- 固定在首页安全区右上。
- 第一枚按钮：`活动规则`
- 第二枚按钮：`预测达人榜`
- 风格为玻璃拟态吸边胶囊，滚动中仍保持可见。

### 5.3 个人战绩摘要条
- 最左：用户头像
- 中部：总预测场次、总命中率、今日表现文案
- 最右：`分享战绩`
- 未参与态显示 `0 场` 与 `命中率 --`，分享入口保持可点击。

### 5.4 日期切换器
- 横向滑动日期胶囊，仅展示日期与简短标签。
- 默认选中当日。
- 无赛事日期置灰不可点。
- 过去日期允许查看已结算结果；未来有赛程日期可预先预测。

### 5.5 比赛卡片结构
1. 顶部信息条：赛事轮次、比赛状态、比赛时间。
2. 对阵信息区：主客队头像、队名、比分或 `VS`、比赛时间入口。
3. 三道预测题：
   - 胜平负
   - 全场进球数
   - 焦点球员能否进球
4. 提交区：状态说明 + 固定文案 `提交预测` 按钮。

### 5.6 焦点球员题表达
- 按“若干名焦点球员”语义设计，可出现 1 组或多组球员条目。
- 默认示例展示两组球员：如 `梅西 VS C 罗`、`阿尔瓦雷斯 VS B 费`
- 若配置不足，则展示降级提示：`本场焦点球员待公布，提交前请完成已展示题目` 的视觉占位方案。

### 5.7 比赛综合内页跳转
- 主客队头像为点击热点。
- 比赛时间区域为点击热点。
- 在 mockup 中通过跳转提示 Toast 演示，不额外发散页面。

## 6. 统一提交模型与状态口径
### 6.1 提交逻辑
- 用户可以自由选择参与哪些比赛。
- 对某场比赛开始作答后，必须完成三道题全部选择。
- 点击 `提交预测` 后，以单场为单位一次性提交。
- 提交成功前不展示投票分布。
- 提交成功后显示投票分布并锁定答案。

### 6.2 `提交预测` 按钮状态
- 可预测且已填满：高亮可点，文案 `提交预测`
- 信息未填全：按钮仍显示 `提交预测`，点击后仅弹 Toast，不提交
- 已提交：按钮置灰禁用，文案 `提交预测`
- 已锁定：按钮置灰禁用，文案 `提交预测`
- 已结算：按钮置灰禁用，文案 `提交预测`
- 已取消：按钮置灰禁用，文案 `提交预测`

### 6.3 Toast 语义
- 未填全：`请完成本场所有预测`
- 超过预测时间：`超过预测时间`
- 提交成功：`提交成功，已锁定本场预测`
- 跳转比赛详情：`打开比赛综合内页示意`

## 7. 关键状态样例
### 7.1 可预测
- 三道题全部可选。
- 示例中已选满三题，CTA 高亮。
- 不展示投票分布。

### 7.2 信息未填全
- 第三题缺失部分选择。
- 点击 `提交预测` 触发未填全 Toast。
- 按钮文案保持不变。

### 7.3 已提交
- 已显示各题投票分布。
- 选项锁定不可修改。
- `提交预测` 置灰。

### 7.4 已锁定
- 比赛已开始，未提交或已提交均不可再修改。
- 卡片显示 `预测已截止 / 等待开奖` 辅助说明。
- `提交预测` 继续置灰。

### 7.5 已结算
- 显示终场比分、每题对错结果、击败百分比。
- `提交预测` 保持置灰，不改文案。

### 7.6 已取消
- 显示比赛取消或球员未出场说明。
- 该场不计入战绩统计。
- `提交预测` 保持置灰，不改文案。

### 7.7 无赛事
- 保留 KV、战绩条、日期条与底部规则卡。
- 中部使用 3D 足球空态插画与氛围文案。
- 页面底部仍用渐变与说明信息填充，避免真空白。

## 8. 达人榜页设计
- 入口来自首页右上角悬浮 `预测达人榜`。
- 头部延续首页高亮赛事视觉，但强调荣誉与金属奖台。
- `每日榜` 支持配合日期条查看单日排行。
- `总榜` 展示活动累计命中题数。
- TOP3 用金银铜奖台立体卡片表现。
- 底部固定 `我的排名` 条，含头像、昵称、名次、命中题数。

## 9. 分享战绩卡设计
- 采用适合系统分享的竖版长图比例。
- 视觉延续首页 KV，但更聚焦“荣誉感”和个人成绩。
- 数据区包含：总预测场次、命中题数、总命中率。
- 文案区显示：`已超越 X% 的预测玩家`。
- 底部带品牌标识与活动二维码占位，引导回流参与。
- 未参与时改为：`0 场 / 0 题 / 命中率 --，快来开启你的世界杯预言`。

## 10. Mockup 覆盖范围
- 活动首页
- 达人榜页
- 分享战绩卡
- 关键状态：可预测、信息未填全、已提交、已锁定、已结算、已取消、无赛事
- 交互演示：规则弹窗示意、达人榜跳转、分享入口、比赛综合内页跳转、未填全 / 超时 / 提交成功 Toast

## 11. 评审关注点
1. 最新 PRD 八项最终口径是否全部可见且无歧义。
2. `提交预测` 是否在所有状态下文案一致。
3. 焦点球员是否按“若干名”语义而非固定双人。
4. H5 视觉是否满足高亮世界杯主题、高阶 3D、无缝衔接、负边距搭叠、玻璃拟态、底部防空白。
5. mockup 是否具备足够清晰的评审交互与关键状态演示。
