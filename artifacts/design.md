# 世界杯每日赛程预测 H5 设计稿说明 v2

## 1. 设计目标
- 为嵌入直播吧 App 的 H5 / mobile-web 活动页提供可评审高保真方案。
- 基于 revised PRD 重建首页、达人榜页、分享战绩卡与关键状态样例。
- 以“世界杯高亮主题 + 高阶 3D 头图 + 玻璃拟态卡片”建立高级赛事氛围，并保证信息识别与提交路径足够清晰。

## 2. 本版必须落实的 revised PRD 更新
- 首页右上角固定悬浮入口：`活动规则`、`预测达人榜`
- 个人战绩摘要条最左侧增加用户头像
- 单场比赛三道题全部完成后，通过 `提交预测` 按钮统一提交
- 点击对阵头像或比赛时间区域可跳转比赛综合内页
- 分享战绩卡重新纳入设计范围
- KV 头图采用高阶 3D 风格建模 / 商业插画表达，并与页面背景无缝衔接

## 3. 信息架构
### 3.1 活动首页
1. 顶部 KV 主视觉区
2. 右上角悬浮入口组
3. 个人战绩摘要条
4. 日期切换器
5. 比赛卡片列表
6. 规则说明与底部氛围延展区

### 3.2 达人榜页
1. 榜单头部
2. `每日榜 / 总榜` Tab
3. TOP3 焦点台
4. TOP4-TOP20 列表
5. 固定底部我的排名条

### 3.3 分享战绩卡
1. 世界杯主题头图
2. 用户头像与昵称
3. 总预测场次 / 命中题数 / 总命中率
4. 超越百分比文案
5. 品牌 Logo 与参与引导

## 4. 视觉系统
### 4.1 主题关键词
- 世界杯高亮
- 赛场灯光
- 高级 3D
- 玻璃拟态
- 奖章质感

### 4.2 颜色
- 背景深色：`#071a18` / `#0d2d27`
- 奖章金：`#f7e08b` / `#d99d36`
- 草地荧绿：`#73f0b2`
- 冰蓝高光：`#8edbff`
- 警示橙：`#ffb25b`
- 结算红：`#ff6a5f`
- 取消灰：`#92a6aa`

### 4.3 材质与层级
- KV 与页面背景使用连续纵深渐变，避免头图与正文断层。
- 摘要条以 `-52px` 负边距搭叠在 KV 下缘，强化海报式层叠。
- 卡片采用半透明白叠层、柔光描边、背景模糊与轻内阴影。
- 按钮采用金色微渐变、内发光边缘、弥散投影，避免纯扁平色块。
- 底部通过环境光、说明卡和长渐变衔接，避免无赛事日出现空白断底。

### 4.4 KV 头图要求
- 视觉主体采用世界杯奖杯 / 足球 / 球场灯架 / 飘带等 3D 组合构图。
- 标题带体积光与描边，允许 10% 到 15% Scrim 提升复杂背景下的可读性。
- 主视觉需表达“今日参与、即刻冲榜”的竞技冲刺感，而非静态海报。

## 5. 首页详细设计
### 5.1 KV 区
- 高度按 390 宽视觉稿控制在约 `320px`
- 文案建议：主标题 `今日赛程预测`，副标题 `答满三题再提交，命中越多排名越高`
- 右上角悬浮入口吸附在安全区内侧，纵向排列两个玻璃按钮

### 5.2 个人战绩摘要条
- 左：用户头像
- 中：`总预测场次`、`总命中率`
- 右：`分享战绩` 按钮
- 未参与默认态：`0 场 | 命中率 --`
- 分享按钮始终可点，未参与时分享卡显示“快来参与预测”

### 5.3 日期切换器
- 水平滑动日期胶囊
- 默认定位当日
- 无赛事日期禁用，不可切换到空数据日期做预测
- 可查看过去日期结算结果和未来已开放赛程

### 5.4 比赛卡片结构
1. 顶部信息行：轮次、状态、赛事时间
2. 对阵信息区：主客队头像、队名、VS / 比分
3. 三道预测题
4. 操作区：提醒文案 + `提交预测` 按钮

### 5.5 比赛详情跳转
- 点击主客队头像
- 点击比赛时间区
- 两者均进入比赛综合内页
- 在 mockup 中以可点击样式与状态提示体现

## 6. 预测交互口径
### 6.1 统一提交模型
- 单场比赛可选择是否参与
- 一旦参与该场，三道题必须全部完成
- 完成后点击 `提交预测`，整场一次性提交
- 成功后才展示全站投票分布，并锁定不可修改

### 6.2 Toast 语义
- 未填全：`请完成本场所有预测`
- 超时：`超过预测时间`
- 提交成功：`提交成功，已锁定本场预测`

### 6.3 提交按钮状态
- 默认：高亮可点击
- 未填全点击：不提交，弹 Toast
- 已截止点击：不提交，弹 Toast
- 成功后：按钮变为 `已提交`

## 7. 关键状态样例
### 7.1 可预测
- 三道题均可点击
- 不显示投票分布
- CTA 为 `提交预测`

### 7.2 信息未填全
- 至少一题未选
- 点击提交后顶部 Toast 提示 `请完成本场所有预测`
- 保持当前草稿态，便于继续补全

### 7.3 已预测
- 三题答案高亮锁定
- 展示各题全站投票分布
- 操作区改为 `已提交`

### 7.4 已锁定
- 未参与用户：展示 `预测已截止`
- 已参与用户：展示 `等待开奖`
- 题目和按钮均不可修改

### 7.5 已结算
- 展示终场比分
- 每题标明我的答案与正确答案
- 显示 `击败了 X% 预测者`

### 7.6 已取消
- 卡片降饱和 + 异常说明
- 文案说明不计入成绩

### 7.7 无赛事
- 空态插画 / 3D 装饰球体
- 保留日期条和底部说明卡
- 用环境背景延长页面高度，避免空白底

## 8. 达人榜页设计
- 入口来自首页右上角悬浮按钮
- 头部沿用世界杯主视觉语言，但更聚焦排行荣誉感
- `每日榜` 可横向模拟查看比赛日，`总榜` 展示活动累计排行
- TOP3 采用奖台式 3D 卡片，金银铜有明显材质区分
- 底部固定条展示 `我的排名 / 头像 / 昵称 / 命中题数`

## 9. 分享战绩卡设计
- 长图比例适合系统分享与保存
- 头部主视觉沿用首页 KV，但弱化交互信息，强化荣誉展示
- 核心数据区包含：总预测场次、命中题数、总命中率
- 排名文案示例：`已超越 88% 的预测玩家`
- 未参与默认卡面文案：`0 场预测，快来开启你的世界杯预言`

## 10. Mockup 演示范围
- 首页
- 达人榜页
- 分享战绩卡
- 关键状态样例：可预测、信息未填全、已预测、已锁定、已结算、已取消、无赛事
- Toast 演示：未填全、超过预测时间、提交成功

## 11. 适配与实现备注
- 视觉宽度按 `390px` 设计，容器真实宽度 `min(100vw, 430px)`
- 需考虑 `safe-area-inset-top/bottom`
- mockup 使用静态数据与前端状态切换演示完整体验
- 所有页面均按 H5 / mobile-web 口径设计，不采用原生 App 导航栏样式

## 12. 评审重点
- revised PRD 六项更新是否全部可见
- `提交预测` 的统一提交逻辑是否足够明确
- 首页悬浮入口、比赛内页跳转、分享卡范围是否表达准确
- 视觉是否满足高阶 3D、玻璃拟态、负边距搭叠、底部防空白四项硬要求
