# 世界杯每日赛程预测活动 Proposal

## 项目背景
世界杯期间，用户对单场比赛结果、进球走势和焦点球员表现有强讨论热情。本项目拟构建一个嵌入直播吧 App 的 H5 活动页，以“每日赛程预测”为核心玩法，通过比赛预测、达人榜、个人战绩与分享能力提升世界杯期间活跃度、留存与比赛详情页导流。

## 目标
- 提升世界杯期间活动页 UV、预测参与率与次日回访率
- 将活动流量导向比赛综合内页，提升比赛内页 UV 与停留时长
- 通过达人榜与个人战绩分享卡增强社交竞争感和传播力

## 目标用户
- 世界杯期间高频关注比赛的球迷用户
- 愿意参与轻互动预测并查看群体分布的用户
- 对排行、赛果反馈和个人战绩有兴趣的活跃内容用户

## 方案概述
### 1. 产品形态
- H5 活动页，嵌入直播吧 App
- 主要面向移动端宿主环境，设计目标为 mobile-web
- 未登录用户可浏览，提交预测时唤起登录

### 2. 核心功能
- 按日期查看每日赛程和比赛预测卡片
- 单场比赛包含三类预测题：胜平负、全场进球数、指定球员是否进球
- 个人战绩摘要条，包含用户头像、总预测场次、总命中率、分享战绩按钮
- 右上角悬浮区提供“活动规则”和“预测达人榜”入口
- 达人榜完整页面支持每日榜 / 总榜切换
- 赛后自动结算，并展示命中结果与击败百分比
- 支持生成个人战绩分享卡

### 3. 已确认业务变化
- 达人榜入口由首页横向卡片改为右上角悬浮按钮
- 比赛详情跳转入口改为点击对阵头像或比赛时间区域
- 分享战绩卡重新纳入范围
- 球员题继续由开发手工配置维护
- 提交规则以 `3.2.3 / 3.3.1` 为准，按单场统一提交
- 焦点球员数量以 `3.2.2` 为准，按“若干名焦点球员”理解，不再固定为每队各 1 名

### 4. 归一化后的提交规则
根据修改后 PRD 的详细交互描述，当前按以下口径收口：

- 用户可自由选择参与哪些比赛
- 但一旦选择某场比赛进行预测，需完成该场 **三道题全部选择**
- 完成后点击 **「提交预测」**，按单场统一提交
- 提交成功后，该场预测进入已预测状态，不可修改

说明：PRD `4.2 第 2 条` 中“可只预测其中 1 题或 2 题”与 `3.2.3 / 3.3.1` 的详细交互冲突，现按更具体的交互描述进行归一化处理，视其为待修正旧文案。

### 5. 数据与接口方向
- 赛程、比赛详情、登录态、分享、排行榜、预测提交、赛后结算需要统一接口约束
- 接口现状仍未最终确认，技术方案阶段默认按“需做接口适配层，且可能需要新增后端能力”准备

## MVP 范围
### 包含
- 活动首页
- 日期切换
- 赛程预测卡片
- 个人战绩摘要
- 悬浮规则入口与悬浮达人榜入口
- 达人榜完整页
- 比赛详情跳转
- 赛后结算结果展示
- 个人战绩分享卡
- 预测异常态（比赛取消、球员未出场、数据延迟）

### 不包含
- 奖品体系
- 复杂后台 CMS（当前默认手工维护球员焦点题）

## 关键风险
- PRD 当前存在核心交互冲突，未澄清前不能继续评审设计
- 现有设计稿基于旧 PRD 产出，已不再完全适配
- 接口现状未确认，可能影响前后端边界与排期
- H5 嵌入 App 需要确认登录、分享、路由跳转等桥接能力

## 建议下一步
- 先确认“单题独立提交”还是“单场三题答完后统一提交”
- 基于澄清后的 PRD 重新确认 proposal/specs
- PRD 重新通过后，再重开 design 阶段并更新高保真设计稿
