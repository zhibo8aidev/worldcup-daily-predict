# 世界杯每日赛程预测活动 Tech Design Tasks

## 1. 任务拆解说明
- 基于已批准 PRD、设计规范、设计说明与 HTML mockup 输出。
- 本拆解采用最终统一口径：
  - 提交规则按 PRD `3.2.3 / 3.3.1`，单场统一提交。
  - 提交按钮文案始终为「提交预测」，仅通过高亮/置灰区分状态。
  - 焦点球员按“若干名焦点球员”语义处理。
  - 首页右上角保留「活动规则 / 预测达人榜」双悬浮入口。
  - 分享战绩卡在范围内。
  - 所有接口未确认项必须显式标注，不默认接口已存在。
- 标记说明：
  - `【待接口确认】`：依赖接口现状、字段、鉴权或错误码确认后才能封板。
  - `【待桥接确认】`：依赖宿主 App JSBridge 或 schema 能力确认。

## 2. 前端

### FE-01 活动首页与全局框架搭建
- 目标：完成 H5 单页活动基础框架，承载 KV、战绩摘要、日期选择器、赛程列表、底部规则承接区。
- 关键产出：页面骨架、通用布局容器、安全区适配、空态与低赛程日防空白布局实现。
- 依赖：`artifacts/design.md`、`artifacts/react_native_mockups/index.html`。
- 风险或待确认项：需确认宿主 WebView 最低兼容范围与安全区处理规范。

### FE-02 首页固定入口与规则弹窗实现
- 目标：实现右上角双悬浮入口，支持活动规则弹窗与达人榜跳转。
- 关键产出：固定悬浮按钮组件、规则弹窗内容容器、滚动过程中的层级与点击态。
- 依赖：FE-01。
- 风险或待确认项：规则文案最终内容、是否由前端静态维护或后端下发需确认；如改为配置化则依赖 `【待接口确认】`。

### FE-03 日期切换与赛程卡片渲染
- 目标：实现日期横向切换、赛程列表排序、无赛事日展示与过去/未来日期浏览。
- 关键产出：日期条组件、赛程卡片渲染器、比赛状态映射、无赛事空态。
- 依赖：FE-01，`【待接口确认】` 赛程查询接口或现有数据源确认。
- 风险或待确认项：需确认日期可选范围、服务端返回时区口径、比赛状态枚举、取消/延期状态码。

### FE-04 单场草稿态与统一提交逻辑
- 目标：实现单场三题草稿态、校验、截止判断、统一提交、提交后锁定与分布展示。
- 关键产出：match-level 状态管理模型、三题联动校验、按钮高亮/置灰规则、Toast 语义、提交成功后不可修改逻辑。
- 依赖：FE-03，`【待接口确认】` 提交接口 payload、提交后回包、投票分布字段、截止时间字段。
- 风险或待确认项：若现有接口仍是单题提交模型，需要前端适配层还是后端改造必须先确认；这是首页主链路关键阻塞项。

### FE-05 焦点球员题动态展示与降级
- 目标：支持“若干名焦点球员”动态渲染，不将题目固定为每队各 1 名。
- 关键产出：焦点球员题组件、单组/多组展示态、配置不足时降级说明、提交前校验口径。
- 依赖：FE-04，配置任务 CFG-01。
- 风险或待确认项：需确认焦点球员配置结构、同场多组球员是否全部必答、未配置球员时该场是否仍要求“三题全部完成”；该项 `【待接口确认】` 或需产品最终补口径。

### FE-06 比赛详情跳转与桥接封装
- 目标：实现点击对阵头像或比赛时间区域进入比赛综合内页，并封装 H5 到 App 的路由能力。
- 关键产出：跳转热点实现、schema/bridge 封装、降级跳转策略、错误兜底提示。
- 依赖：FE-03，`【待桥接确认】` 比赛详情 schema 规则或路由 API。
- 风险或待确认项：需确认不同端统一 URL/schema、未登录或 bridge 不可用时的降级方式。

### FE-07 达人榜页面实现
- 目标：实现达人榜完整页，支持每日榜/总榜切换、TOP3 特殊样式、TOP4-TOP20 列表、底部我的排名条。
- 关键产出：榜单页路由/弹层承载方案、榜单 Tab、日期筛选、我的排名固定条。
- 依赖：FE-02，`【待接口确认】` 排行榜查询接口、我的排名接口、榜单刷新时机。
- 风险或待确认项：需确认每日榜日期切换接口是否独立、同分排序字段是否直接返回。

### FE-08 个人战绩摘要与分享战绩卡
- 目标：实现顶部个人战绩摘要条与分享战绩卡生成链路。
- 关键产出：战绩摘要展示、分享卡视图、长图生成方案、未参与态展示、分享触发入口。
- 依赖：FE-01，`【待接口确认】` 用户战绩数据接口，`【待桥接确认】` 系统分享/保存相册能力。
- 风险或待确认项：需确认分享卡数据字段、二维码来源、分享回调是否要求埋点闭环。

### FE-09 全局状态、异常态与视觉状态收口
- 目标：统一整理可预测、未填全、已提交、已锁定、已结算、已取消、无赛事等状态在前端的展示与迁移规则。
- 关键产出：前端状态映射表、异常态展示策略、统一按钮态与辅助文案口径、错误页/Toast 策略。
- 依赖：FE-03、FE-04、FE-05、FE-07、FE-08。
- 风险或待确认项：需确认比赛取消、球员未出场、数据延迟的服务端返回口径；`【待接口确认】`。

## 3. 后端

### BE-01 接口现状盘点与差异评估
- 目标：确认赛程、预测提交、投票分布、排行榜、战绩、分享、规则等能力是否已有可复用接口。
- 关键产出：接口盘点表、字段差异清单、鉴权方式、错误码与限流约束清单。
- 依赖：无。
- 风险或待确认项：该任务是多项前端任务封板前置；若无现成接口，需尽快转入新增接口设计。

### BE-02 赛程与比赛状态聚合接口
- 目标：提供支持首页日期切换与比赛卡片渲染的统一数据源。
- 关键产出：赛程聚合接口或适配层、比赛状态标准枚举、开赛截止时间、比分/结算字段、焦点球员配置透出字段。
- 依赖：BE-01。
- 风险或待确认项：若上游赛程源无法直接提供前端所需状态，需要新增聚合逻辑；当前 `【待接口确认】`。

### BE-03 单场统一提交与投票分布能力
- 目标：提供 match-level 统一提交接口，并在提交成功后返回或支持查询各题投票分布。
- 关键产出：统一提交 API、幂等与重复提交保护、截止校验、回包结构、投票分布查询策略。
- 依赖：BE-01。
- 风险或待确认项：若现有系统仅支持单题提交，需要新增接口或服务端编排；这是主链路最高优先级 `【待接口确认】`。

### BE-04 达人榜与个人战绩接口
- 目标：提供每日榜、总榜、我的排名、个人战绩摘要、分享卡数据的统一查询能力。
- 关键产出：排行榜接口、我的排名补位逻辑、个人战绩接口、分享卡 percentile 计算口径。
- 依赖：BE-01。
- 风险或待确认项：需确认同分排序规则是否由服务端完全产出；分享卡“超越 X%”的算法口径需明确。

### BE-05 规则、配置与异常态数据支撑
- 目标：支撑规则文案、焦点球员配置、比赛取消/球员未出场等异常态的标准化输出。
- 关键产出：规则文案来源方案、焦点球员配置结构、异常态判定字段与结算排除逻辑。
- 依赖：BE-01。
- 风险或待确认项：若继续采用人工配置，需要明确配置落点、发布流程与责任人；若改接口下发则为 `【待接口确认】`。

## 4. 配置与运营支持

### CFG-01 焦点球员手工配置方案落地
- 目标：明确焦点球员由开发手工维护时的数据模板、更新频次、校验规则与生效流程。
- 关键产出：配置模板、字段说明、配置校验清单、缺配时降级策略。
- 依赖：BE-05、FE-05。
- 风险或待确认项：需确认“若干名焦点球员”最小/最大配置数量，以及同场多组球员是否全部纳入提交校验。

### CFG-02 活动规则与分享素材运营准备
- 目标：准备规则弹窗文案、分享卡文案、二维码/品牌素材、榜单展示昵称规范。
- 关键产出：规则文案终稿、分享卡静态素材包、文案字段清单。
- 依赖：FE-02、FE-08。
- 风险或待确认项：若规则或素材改为接口下发，则依赖 `【待接口确认】` 的配置接口或 CMS 方案。

### CFG-03 宿主桥接能力确认清单
- 目标：与 App 侧确认登录、分享、页面跳转、埋点透传等桥接能力现状。
- 关键产出：JSBridge 能力清单、调用参数说明、端差异说明、降级方案表。
- 依赖：无。
- 风险或待确认项：该任务直接影响 FE-06、FE-08、DT-03 与联调排期；属 `【待桥接确认】`。

## 5. 埋点

### DT-01 首页与核心浏览埋点设计
- 目标：覆盖活动页曝光、日期切换、比赛卡曝光、空态曝光等浏览链路。
- 关键产出：埋点事件表、字段字典、前端触发时机定义。
- 依赖：FE-01、FE-03。
- 风险或待确认项：需确认宿主埋点通道与公共字段要求；`【待桥接确认】`。

### DT-02 预测与榜单交互埋点设计
- 目标：覆盖题目选择、提交点击、提交成功/失败、规则弹窗、达人榜进入、Tab 切换等关键交互。
- 关键产出：预测链路埋点方案、失败原因枚举、榜单交互埋点表。
- 依赖：FE-02、FE-04、FE-07、BE-03。
- 风险或待确认项：需确认“成功埋点”以提交成功为准，避免按单题点击误记成功；失败码来源依赖 `【待接口确认】`。

### DT-03 分享与桥接回调埋点设计
- 目标：覆盖分享战绩卡生成、分享渠道拉起、分享成功/失败回调、比赛详情跳转。
- 关键产出：分享埋点方案、bridge 回调映射、比赛详情跳转埋点。
- 依赖：FE-06、FE-08、CFG-03。
- 风险或待确认项：若宿主不返回分享结果，需要定义可接受的替代统计口径；`【待桥接确认】`。

## 6. 联调与验收

### QA-01 接口联调计划与 Mock 策略
- 目标：在接口现状未完全确认前，制定前后端联调顺序、Mock 数据范围与阻塞升级机制。
- 关键产出：联调顺序表、Mock 覆盖清单、接口确认检查点。
- 依赖：BE-01。
- 风险或待确认项：统一提交接口与排行榜接口若延期，将直接阻塞核心验收路径。

### QA-02 关键业务链路验收
- 目标：验收首页浏览、单场三题统一提交、提交后锁定、投票分布展示、榜单切换、分享战绩卡主链路。
- 关键产出：业务验收用例、状态覆盖矩阵、缺陷记录模板。
- 依赖：FE-04、FE-07、FE-08、BE-02、BE-03、BE-04。
- 风险或待确认项：需按最终接口回包核对按钮状态、锁定时机与异常提示。

### QA-03 异常态与兼容性验收
- 目标：验收比赛取消、球员未出场、数据延迟、登录失效、bridge 不可用、无赛事日等场景。
- 关键产出：异常态测试清单、端兼容清单、降级行为确认结果。
- 依赖：FE-09、CFG-03、BE-05。
- 风险或待确认项：需覆盖 iOS / Android / HarmonyOS 宿主差异；bridge 降级必须实机验证。

### QA-04 上线前联合评审与发布检查
- 目标：在 tech_review 后形成可执行排期与上线前检查项。
- 关键产出：前后端发布依赖表、配置上线清单、验收结论、待关闭风险列表。
- 依赖：全部任务。
- 风险或待确认项：若 BE-01 或 CFG-03 未完成，不能判定排期与上线风险已收敛。

## 7. 接口现状确认依赖汇总
- 直接依赖“接口现状确认”的任务：FE-03、FE-04、FE-05、FE-07、FE-08、FE-09、BE-02、BE-03、BE-04、BE-05、DT-02、QA-01。
- 直接依赖“桥接能力确认”的任务：FE-06、FE-08、CFG-03、DT-01、DT-03、QA-03。
- 优先级最高的待确认项：
  - 是否已有支持单场统一提交的预测接口。
  - 赛程/比赛状态/结算状态的字段与枚举口径。
  - 焦点球员配置结构与未配置时的提交口径。
  - 达人榜与个人战绩是否已有可复用接口。
  - App 是否提供登录、分享、路由跳转、埋点透传的统一桥接能力。

## 8. 建议评审顺序
1. 先审最终口径是否与 PRD/设计稿完全一致，尤其是“单场统一提交”“按钮文案恒定”“若干名焦点球员”。
2. 再审 BE-01 与 CFG-03 的确认路径，因为它们决定前后端边界与排期可信度。
3. 最后按首页主链路、榜单、分享、埋点、联调验收的顺序确认任务完整性。
