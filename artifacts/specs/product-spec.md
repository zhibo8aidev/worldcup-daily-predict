# Product Spec - 世界杯每日赛程预测活动

## 1. 产品概述
一个嵌入直播吧 App 的世界杯每日赛程预测 H5。用户按比赛日查看赛程，并对单场比赛进行预测，赛后自动结算，查看个人战绩、达人榜，并可生成分享战绩卡。

## 2. 业务目标
- 提升世界杯期间活动页活跃度与次日回访率
- 拉动比赛综合内页跳转与浏览时长
- 通过排行榜和分享卡增强传播与社交竞争感

## 3. 页面范围
### 3.1 活动首页
- 顶部 KV / 活动标题区
- 个人战绩摘要条（头像、总预测场次、总命中率、分享战绩按钮）
- 右上角悬浮入口区（活动规则、预测达人榜）
- 日期选择器
- 当日赛程预测卡片列表

### 3.2 达人榜页面
- 每日榜 / 总榜 Tab
- TOP20 排行列表
- 固定底部“我的排名”条

### 3.3 分享战绩能力
- 从个人战绩摘要条进入
- 生成长图并支持系统分享 / 保存相册

## 4. 比赛卡片结构
- 对阵信息：主客队队徽、队名、开赛时间、赛事轮次
- 交互跳转：点击对阵头像或比赛时间区域进入比赛综合内页
- 预测题一：胜平负
- 预测题二：全场进球数
- 预测题三：指定球员能否进球（主队、客队各 1 人；支持异常降级）
- 提交预测按钮

## 5. 当前确认的规则变化
- 达人榜入口改为右上角悬浮按钮
- 活动规则入口改为右上角悬浮按钮
- 分享战绩卡重新纳入范围
- 球员焦点题继续由开发手工配置维护

## 6. 归一化后的提交规则
当前按 PRD 中更详细的交互描述收口为：

- 用户可自由选择参与哪些比赛
- 单场比赛内，必须完成三道题后，点击 **「提交预测」** 统一提交
- 提交成功前不显示投票分布
- 提交成功后显示该场各题投票分布，并锁定不可修改

说明：`4.2 第 2 条` 中“每道预测题独立参与，可只预测 1 题或 2 题”与 `3.2.3 / 3.3.1` 的详细交互冲突，现以详细交互描述为准，将该条视为待修订旧文案。

## 7. 其他规则与结算
- 胜平负按常规 90 分钟结果判定
- 进球数按常规 90 分钟统计
- 球员进球按国际足联官方统计，乌龙球不算
- 开赛后不可继续预测
- 比赛取消 / 推迟 / 球员未出场时按 PRD 定义的取消规则处理

## 9. 数据配置
### 9.1 赛程基础数据
需要比赛 ID、主客队信息、开赛时间、轮次、比赛状态、最终比分。

### 9.2 球员焦点题配置
- 当前按开发手工配置维护
- 每场默认主队、客队各 1 名焦点球员
- 若配置不全，按 PRD 规则降级展示

## 10. 埋点与分享
- 保留活动页、预测、日期切换、达人榜、分享战绩相关埋点
- 分享卡生成、分享渠道、分享成功回调均在范围内

## 11. 非功能要求
- H5 需适配 iOS / Android / HarmonyOS 宿主 App 浏览环境
- 页面视觉需符合用户提供的世界杯高亮主题设计规范
- 页面底部在低赛程日不得出现大面积空白
- 关键状态展示需清晰、可快速理解
