# Reviews

Store one record per approval or rejection event.

Suggested naming:

- `prd_review-YYYYMMDD-HHMM.md`
- `design_review-YYYYMMDD-HHMM.md`
- `tech_review-YYYYMMDD-HHMM.md`
- `release_review-YYYYMMDD-HHMM.md`

Each record should include:

- project id
- stage
- decision
- raw user message
- impact
- next stage
- artifact readiness check
- owner / handoff / run validation summary
